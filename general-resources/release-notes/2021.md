# 2021

## 2021/12/30

You can now use [Global Logistics](../../using-our-services/global-logistics.md), an end-to-end cross-border delivery management solution.

## 2021/12/14

* When [sending product data in checkouts](../../integration-options/checkouts/creating-checkouts/describing-the-items/#sending-product-data), you can now use [`productDetails`](../../product-management/using-product-details.md).
* When Digital River receives a [chargeback request](../../order-management/returns-and-refunds-1/disputes-and-chargebacks.md), we now move the associated order's [`state`](../../order-management/orders/the-order-lifecycle.md#order-states-and-events) to `dispute`.

## 2021/12/13

{% hint style="warning" %}
**Breaking changes**

In [version](../versioning.md) `2021-12-13`, the following changes have been deployed:

* In [registered checkouts](../../integration-options/checkouts/creating-checkouts/using-the-checkout-identifier.md#registered-checkouts-or-invoices), Digital River no longer by default uses the associated [customerâ€™s](https://www.digitalriver.com/docs/digital-river-api-reference/#tag/Customers) data to populate certain undefined [checkout](https://www.digitalriver.com/docs/digital-river-api-reference/#tag/Checkouts) attributes. For each [attribute common to both customers and checkouts](../../integration-options/checkouts/creating-checkouts/using-the-checkout-identifier.md#common-attributes), you must now define that attribute in checkouts.
* Digital River has removed [`allowOversell`](../../product-management/creating-and-updating-skus.md#allow-oversell) from SKUs.
{% endhint %}

## 2021/12/7

We made the following changes to [Digital River Dashboard](https://dashboard.digitalriver.com):

* You can now add, edit, and delete a fee from the [SKU details](../../administration/dashboard/catalog/skus/viewing-the-sku-details.md) page.
* You can now view the metadata associated with an order from the [Order details](../../administration/dashboard/order-management/orders/viewing-the-order-details.md) page. \*\*\*\*
* The [Checkout details](../../administration/dashboard/order-management/checkouts/viewing-the-checkout-details.md) page now displays the full checkout details.

## 2021/11/22

We added the ability to [download a credit memo](https://docs.digitalriver.com/digital-river-api/administration/dashboard/order-management/orders/downloading-a-credit-memo) from the Dashboard.

## 2021/10/25

You're no longer allowed to [attach non-reusable sources to customers](../../payments/payment-sources/using-the-source-identifier.md#attaching-sources-to-customers).

## 2021/10/19

You can now subscribe to the [`order.cancelled`](../../order-management/creating-and-updating-an-order.md#the-order-cancelled-event) event.

## 2021/10/15

Every physical SKU that is shipped cross-border into Switzerland, must now contain a `weight` and `weightUnit`. For details, refer to the [weight and weightUnit](../../product-management/creating-and-updating-skus.md#weight-and-weight-unit) section on the [Managing SKUs](../../product-management/creating-and-updating-skus.md) page.

## 2021/10/8

We added the ability to the ability to [manage customers](../../administration/dashboard/customers/) from the [Dashboard](https://dashboard.digitalriver.com). You can also [attach a tax certificate](../../administration/dashboard/customers/tax-ids/adding-a-tax-certificate.md) or [tax ID](../../administration/dashboard/customers/tax-ids/adding-a-tax-id.md) to a customer.

## 2021/10/4

* We added a new [legal entity](../../integration-options/checkouts/creating-checkouts/selling-entities.md), Digital River UK Limited.
* In [`POST/checkouts`](https://www.digitalriver.com/docs/digital-river-api-reference/#operation/createCheckouts) and [`POST/checkouts/{id}`](https://www.digitalriver.com/docs/digital-river-api-reference/#operation/updateCheckouts) requests, you can now include an `organization.id`.

## 2021/10/1

We added the ability to [fulfill an order either partially or fully](../../administration/dashboard/order-management/orders/fulfilling-items.md) and [cancel a fulfillment order either fully or partially](../../administration/dashboard/order-management/orders/cancelling-items.md) from the [Dashboard](https://dashboard.digitalriver.com/login).

## 2021/9/10

We now provide [test tax identifiers](../../integration-options/checkouts/creating-checkouts/tax-identifiers.md#test-tax-identifiers).

## 2021/9/9

* We now conduct enhanced [validation of SKU group identifiers](../../product-management/setting-up-sku-groups.md#unique-identifier) .
* We created new [rules for deleting SKU groups](../../product-management/setting-up-sku-groups.md#deleting-sku-groups).
* You can now [use query parameters to return a list of SKU groups](https://www.digitalriver.com/docs/digital-river-api-reference/#operation/listSkuGroups).
* You can now [search for SKUs by `skuGroupId`](https://www.digitalriver.com/docs/digital-river-api-reference/#operation/listSkus).

## 2021/9/1

We expanded the pay later options that are now available in the [PayPal Wallet](../../payments/payment-integrations-1/digitalriver.js/payment-methods/paypal.md#paypal-express-checkout-digital-wallet), including [PayPal RatenZahlung](../../payments/payment-integrations-1/digitalriver.js/payment-methods/paypal.md#paypal-ratenzahlung) for German shoppers.

## 2021/8/25

You can now [use `applicability` to filter tax identifiers](../../integration-options/checkouts/creating-checkouts/tax-identifiers.md#filtering-applicable-tax-identifiers).

## 2021/8/24

Digital River now supports the use of [SEPA Direct Debit with Store Credit](../../payments/payment-sources/using-the-source-identifier.md#combining-primary-and-secondary-payment-sources).

## 2021/8/17

You can now [group products](../../product-management/setting-up-sku-groups.md) that share a basic Harmonized System code and are exported to the same set of countries.

## 2021/8/13

You can now use [Digital River's subscription service](../../integration-options/checkouts/subscriptions/digital-river-coordinated-subscriptions.md) to schedule and process recurring payments.

## 2021/8/6

We added `supportsStorage` to indicate whether a payment method supports storage to [Retrieving available payment methods](../../payments/payment-integrations-1/digitalriver.js/reference/digitalriver-object.md#retrieving-available-payment-methods).

## 2021/7/29

We expanded the pay later options that are now available in the [PayPal Wallet](../../payments/payment-integrations-1/digitalriver.js/payment-methods/paypal.md#paypal-express-checkout-digital-wallet), including[ PayPal Pay in 4](../../payments/payment-integrations-1/digitalriver.js/payment-methods/paypal.md#paypal-pay-in-4) for shoppers in Australia and France, as well as [PayPal Pay in 3](../../payments/payment-integrations-1/digitalriver.js/payment-methods/paypal.md#paypal-pay-in-3) and [PayPal Credit](../../payments/payment-integrations-1/digitalriver.js/payment-methods/paypal.md#paypal-credit) for UK shoppers.

## 2021/7/27

You can now use the [Delayed Payment Instructions element](../../developer-resources/reference/elements/delayed-payment-instructions-element.md) to generate a template that contains the instructions that explain how to complete payment for delayed payment types such as Konbini and Wire Transfer.

## 2021/7/22

Digital River now supports the use of [Online Banking with Store Credit](../../payments/payment-sources/using-the-source-identifier.md#combining-primary-and-secondary-payment-sources).

## 2021/7/6

We added the ability to [manage payment methods](../../administration/dashboard/settings/payment-methods/). You can now[ view payment methods](../../administration/dashboard/settings/payment-methods/viewing-your-payment-methods.md) and their [details](../../administration/dashboard/settings/payment-methods/viewing-payment-method-details/), [enable currencies](../../administration/dashboard/settings/payment-methods/viewing-payment-method-details/enabling-currencies.md), [manage countries](../../administration/dashboard/settings/payment-methods/viewing-payment-method-details/managing-countries.md), and [disable a payment method](../../administration/dashboard/settings/payment-methods/disabling-a-payment-method.md).

## 2021/7/2

We added support for [Pay in 3](../../payments/payment-integrations-1/digitalriver.js/payment-methods/paypal.md#paypal-pay-in-3) to the [PayPal ](../../payments/payment-integrations-1/digitalriver.js/payment-methods/paypal.md#paypal)payment method.

## 2021/6/28

* You can now [add](../../administration/dashboard/catalog/skus/adding-a-sku.md), [edit](../../administration/dashboard/catalog/skus/editing-a-sku.md), and [delete a SKU](../../administration/dashboard/catalog/skus/deleting-a-sku.md) from the [Dashboard](https://dashboard.digitalriver.com/login).
* You can now [accept a return](../../administration/dashboard/order-management/orders/accepting-a-return.md) from the [Dashboard](https://dashboard.digitalriver.com/login).

## 2021/6/22

You can now use [`POST/invoices/{id}/sources/{sourceId}`](https://www.digitalriver.com/docs/digital-river-api-reference/#operation/attachSourceToInvoice) and [`DELETE/invoices/{id}/sources/{sourceId}`](https://www.digitalriver.com/docs/digital-river-api-reference/#operation/detachSourceToInvoice) to attach and detach sources from invoices.

## 2021/6/17

Digital River now supports the use of [bPay with Store Credit](../../payments/payment-sources/using-the-source-identifier.md#combining-primary-and-secondary-payment-sources).

## 2021/6/11

* Using the [Files API](https://www.digitalriver.com/docs/digital-river-api-reference/#tag/Files), the [maximum file size](../../order-management/files-and-file-links-1/files.md#file-and-purpose) you can upload to Digital River's servers is now 10 MB.
* You can now [search](../../administration/dashboard/catalog/skus/searching-for-skus.md), [filter](../../administration/dashboard/catalog/skus/filtering-your-skus.md), and [view ](../../administration/dashboard/catalog/skus/viewing-the-sku-details.md)your SKUs on the [Dashboard](https://dashboard.digitalriver.com/login).

## 2021/6/7

Digital River now supports the use of [Konbini with Store Credit](../../payments/payment-sources/using-the-source-identifier.md#combining-primary-and-secondary-payment-sources).

## 2021/6/3

In the [Invoices](https://www.digitalriver.com/docs/digital-river-api-reference/#tag/Invoices) resource, you can now [access details on product-level fee amounts](../../integration-options/checkouts/creating-checkouts/accessing-regulatory-fee-information.md#getting-information-on-specific-fees).

## 2021/6/1

We added OAuth as an additional authentication method when you [create ](../../order-management/events-and-webhooks-1/webhooks/creating-a-webhook.md)or [edit a webhook](../../administration/dashboard/developers/webhooks/editing-a-webhook.md) in the Dashboard.

## 2021/5/17

Digital River now supports [Alipay with Store Credit](../../payments/payment-sources/using-the-source-identifier.md#combining-primary-and-secondary-payment-sources) allowing you to offer Alipay with store credit on an order.

## 2021/5/13

You can no longer issue order-level fee refunds. In addition to standard refunds, the types of order-level and product-level refunds we support are listed [here](../../order-management/returns-and-refunds-1/refunds/issuing-refunds.md#specifying-the-type-of-refund).

## 2021/4/29

You can now include [bill to information](../../integration-options/checkouts/creating-checkouts/providing-address-information.md#bill-to-address) in a `POST/invoices` request. Additionally, when we give you back an Invoice, it now contains a `billTo` value. This is true even when you did not send `billTo` in the payload of a `POST/invoices`.

## 2021/4/28

The following physical attributes have been added to the Sales summaries and Transactions in Dashboard: client upstream ID, payment type, ship from country, ship to country, bill to country, and SKU tax code. These attributes appear when you [export transactions](../../administration/dashboard/finance/transactions/exporting-your-transactions.md) or [sales summary details](../../administration/dashboard/finance/sales-summaries/exporting-your-sales-summary-details..md) in the [Finance ](../../administration/dashboard/finance/)section.

## 2021/4/27

You can now [access details on product-level fee amounts](../../integration-options/checkouts/creating-checkouts/accessing-regulatory-fee-information.md#getting-information-on-specific-fees).

## 2021/4/21

For troubleshooting purposes, we have implemented the following HTTP request headers: [`upstream-id`](../../developer-resources/api-structure.md#upstream-identifier), [`upstream-session-id`](../../developer-resources/api-structure.md#upstream-session-identifier), and [`upstream-application-id`](../../developer-resources/api-structure.md#upstream-application-identifier). As a [best practice](../../developer-resources/best-practices.md#call-debugging), we recommend that you include these headers in your requests.

## 2021/4/13

You can now [configure SKUs for Digital River managed fulfillments](../../product-management/creating-and-updating-skus.md#managed-fulfillments). Once properly configured, you can then use them in the [Digital River fulfillment model](../../order-management/fulfillments.md#automated-fulfillment-model).

## 2021/3/23

{% hint style="warning" %}
**Breaking changes**

In version `2021-03-23`, the [Checkout](https://www.digitalriver.com/docs/digital-river-api-reference/#tag/Checkouts) and [Order](https://www.digitalriver.com/docs/digital-river-api-reference/#tag/Orders) resources now return the first-class `payment` attribute. This attribute contains the array of applied payment sources, the array of charges created from those sources, and additional payment session information. You can use this new session information to [determine when to convert a checkout to an order](../../integration-options/checkouts/creating-checkouts/payment-sessions.md#how-to-determine-when-to-create-an-order).
{% endhint %}

* We now provide you the ability to:
  * [Create secondary payment sources](../../payments/payment-sources/using-the-source-identifier.md#creating-secondary-sources) with `POST/sources`.
  * Attach [multiple secondary sources](../../payments/payment-sources/using-the-source-identifier.md#attaching-sources-to-checkouts) to a checkout.
  * Use [`POST/checkouts/{id}/sources/{sourceId}`](../../payments/payment-sources/using-the-source-identifier.md#passing-the-source-identifier-as-a-path-parameter) to attach a source to a checkout.
  * Use [`DELETE/checkouts/{id}/sources/{sourceId}`](../../payments/payment-sources/using-the-source-identifier.md#detaching-sources-from-checkouts) to detach a source from a checkout.
* When [combining primary and secondary sources](../../payments/payment-sources/using-the-source-identifier.md#combining-primary-and-secondary-payment-sources), you should be aware of how to sequence their application in both guest and registered checkout workflows.

## 2021/3/11

Added the `continue` type to the list of button types for [Drop-in](../../payments/payment-integrations-1/drop-in/) and made it the default type in [Customizing the text of the Drop-in button](../../payments/payment-integrations-1/drop-in/drop-in-integration-guide.md#customizing-the-text-of-the-drop-in-button).

## 2021/3/10

When [sequencing address information in your checkout workflows](../../integration-options/checkouts/creating-checkouts/providing-address-information.md#sequencing-address-information-in-checkout-workflows), you can now use [`billTo`](../../integration-options/checkouts/creating-checkouts/providing-address-information.md#bill-to-address) to obtain an accurate tax calculation before you attach a payment source to the checkout.

## 2021/2/23

{% hint style="warning" %}
**Breaking changes**

We made the following changes in version `2021-02-23`:

* You can now access the [tax identifiers resource](../../integration-options/checkouts/creating-checkouts/tax-identifiers.md).
* Tax identifiers must be created using the Tax Identifiers API. You can no longer use the Customers API to perform this operation.
* Once you've upgraded, any tax identifiers you previously created with the Customers API will no longer be available. Please contact us to have your data migrated.
{% endhint %}

## 2021/2/16

For [Digital River coordinated fulfillments](../../order-management/fulfillments.md#digital-river-coordinated-fulfillments), you should now send [shipment identifiers](../../order-management/informing-digital-river-of-a-fulfillment.md#shipment-identifiers) in `POST/fulfillments` requests.

## 2021/1/18

You can now use the Digital River API to [manage an order's physical fulfillment](../../integration-options/checkouts/handling-digital-river-coordinated-fulfillments/global-fulfillments.md).

## 2021/1/15

You can now use the [`authenticateSource` method](../../payments/payment-integrations-1/digitalriver.js/reference/digitalriver-object.md#authenticating-sources) to determine whether the saved payment source selected by a customer during the checkout process requires [Strong Customer Authentication (SCA)](../../payments/psd2-and-sca/).

## 2021/1/11

* When creating a source, you can now configure the `usage` parameter. It allows you to specify how the type of transaction the source will likely be used in the future. The parameter is available in both [Drop-in](../../payments/payment-integrations-1/drop-in/drop-in-integration-guide.md#specifying-a-sources-future-use) and [DigitalRiver.js](../../payments/payment-integrations-1/digitalriver.js/reference/digitalriver-object.md#specifying-a-sources-future-use). We recommend you use it when building [purchase](../../integration-options/checkouts/building-you-workflows/#credit-card-details-saved-by-customer-during-checkout) and [account management](../../integration-options/checkouts/building-you-workflows/#account-management-flows) workflows that allow customers to save their credit card information.
* When [creating an instance of Drop-in](../../payments/payment-integrations-1/drop-in/drop-in-integration-guide.md#step-6-allow-the-shopper-to-interact-with-hydrate), you can now [customize the text of the button](../../payments/payment-integrations-1/drop-in/drop-in-integration-guide.md#customizing-the-text-of-the-drop-in-button). Additionally, we created new [Drop-in options](../../payments/payment-integrations-1/drop-in/drop-in-integration-guide.md#drop-in-options) that allow you to set the `showTermsOfSaleDisclosure` and `disableAutomaticRedirects` parameters.
